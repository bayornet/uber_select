This example uses optgroups to separate option types. Options and optgroups may have a data-visibility attribute that determines when the option is available for matching. A value of no-query means the option is only visible when the user isn't performing a search. A value of query means the option is only visible during a search. If an option doesn't specify this value, it will inherit it from its optgroup.
